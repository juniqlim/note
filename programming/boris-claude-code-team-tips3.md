# Boris Cherny의 Claude Code 팀 생산성 팁 10가지
https://x.com/bcherny/status/2017742741636321619

저는 Boris이고 Claude Code를 만들었습니다. Claude Code 팀에서 직접 얻은 몇 가지 팁을 빠르게 공유하고 싶었습니다. 팀이 Claude를 사용하는 방식은 제가 사용하는 방식과 다릅니다. 기억하세요: Claude Code를 사용하는 정답은 없습니다 -- 모든 사람의 설정이 다릅니다. 자신에게 맞는 것을 찾기 위해 실험해보세요!

## 1. 더 많이 병렬로 하기

3-5개의 git worktree를 동시에 띄우고, 각각 별도의 Claude 세션을 병렬로 실행하세요. 이것이 가장 큰 생산성 향상이며, 팀의 최고 팁입니다. 개인적으로 저는 여러 git checkout을 사용하지만, Claude Code 팀 대부분은 worktree를 선호합니다 -- @amorriscode가 Claude Desktop 앱에 이를 위한 네이티브 지원을 구축한 이유이기도 합니다!

어떤 사람들은 worktree에 이름을 붙이고 셸 별칭(za, zb, zc)을 설정해서 한 번의 키 입력으로 이동합니다. 또 다른 사람들은 로그 읽기와 BigQuery 실행만을 위한 전용 "analysis" worktree를 가지고 있습니다.

참고: https://code.claude.com/docs/en/common-workflows#run-parallel-claude-code-sessions-with-git-worktrees

## 2. 모든 복잡한 작업은 plan 모드로 시작하기

계획에 에너지를 쏟으면 Claude가 구현을 한 번에 해냅니다.

한 사람은 Claude가 계획을 작성하게 한 다음, 두 번째 Claude를 띄워 스태프 엔지니어로서 리뷰하게 합니다.

또 다른 사람은 뭔가 잘못되는 순간, plan 모드로 돌아가서 다시 계획한다고 합니다. 계속 밀어붙이지 마세요. 그들은 또한 빌드뿐만 아니라 검증 단계에서도 Claude에게 명시적으로 plan 모드에 들어가라고 말합니다.

## 3. CLAUDE.md에 투자하기

매번 수정 후에는 이렇게 끝내세요: "CLAUDE.md를 업데이트해서 다시는 그 실수를 하지 않도록 해." Claude는 자신을 위한 규칙을 작성하는 데 놀라울 정도로 뛰어납니다.

시간이 지남에 따라 CLAUDE.md를 가차 없이 편집하세요. Claude의 실수율이 측정 가능하게 떨어질 때까지 계속 반복하세요.

한 엔지니어는 Claude에게 모든 작업/프로젝트에 대한 notes 디렉토리를 유지하게 하고, 매 PR 후에 업데이트하게 합니다. 그런 다음 CLAUDE.md가 그것을 가리키게 합니다.

## 4. 자신만의 skill을 만들고 git에 커밋하기

모든 프로젝트에서 재사용하세요.

팀의 팁:
- 하루에 한 번 이상 하는 일이라면, skill이나 command로 만드세요
- /techdebt 슬래시 커맨드를 만들고 매 세션 끝에 실행해서 중복된 코드를 찾아 제거하세요
- 7일간의 Slack, GDrive, Asana, GitHub를 하나의 컨텍스트 덤프로 동기화하는 슬래시 커맨드를 설정하세요
- dbt 모델을 작성하고, 코드를 리뷰하고, dev에서 변경 사항을 테스트하는 분석 엔지니어 스타일의 에이전트를 만드세요

자세히 보기: https://code.claude.com/docs/en/skills#extend-claude-with-skills

## 5. Claude가 대부분의 버그를 스스로 고칩니다

이렇게 합니다:

Slack MCP를 활성화한 다음, Slack 버그 스레드를 Claude에 붙여넣고 그냥 "fix"라고 말하세요. 컨텍스트 전환이 전혀 필요 없습니다.

또는, "실패하는 CI 테스트 고쳐"라고 말하세요. 어떻게 할지 세세하게 관리하지 마세요.

분산 시스템 문제 해결을 위해 Claude에게 docker 로그를 가리키세요 -- 이 부분에서 놀라울 정도로 능숙합니다.

## 6. 프롬프팅 수준 높이기

a. Claude에게 도전하세요. "이 변경 사항들에 대해 나를 심문하고, 내가 네 테스트를 통과할 때까지 PR을 만들지 마"라고 말하세요. Claude를 리뷰어로 만드세요. 또는, "이게 작동한다는 걸 증명해봐"라고 말하고 Claude가 main과 feature 브랜치 사이의 동작 차이를 비교하게 하세요.

b. 평범한 수정 후에는 이렇게 말하세요: "지금까지 알게 된 모든 것을 바탕으로, 이걸 버리고 우아한 솔루션을 구현해"

c. 작업을 넘기기 전에 상세한 스펙을 작성하고 모호함을 줄이세요. 구체적일수록 결과물이 좋아집니다.

## 7. 터미널 및 환경 설정

팀은 Ghostty를 좋아합니다! 여러 사람이 동기화된 렌더링, 24비트 컬러, 적절한 유니코드 지원을 좋아합니다.

Claude를 더 쉽게 관리하려면, /statusline을 사용해서 status bar가 항상 컨텍스트 사용량과 현재 git 브랜치를 표시하도록 커스터마이즈하세요. 우리 중 많은 사람들이 터미널 탭에 색상 코드와 이름을 붙이고, 때로는 tmux를 사용합니다 — 작업/worktree당 하나의 탭.

음성 받아쓰기를 사용하세요. 타이핑보다 3배 빠르게 말할 수 있고, 결과적으로 프롬프트가 훨씬 더 상세해집니다. (macOS에서 fn x2)

더 많은 팁: https://code.claude.com/docs/en/terminal-config

## 8. 서브 에이전트 사용하기

a. Claude가 더 많은 컴퓨팅을 투입하길 원하는 요청에는 "use subagents"를 추가하세요

b. 개별 작업을 서브 에이전트에 맡겨서 메인 에이전트의 컨텍스트 윈도우를 깨끗하고 집중된 상태로 유지하세요

c. 권한 요청을 훅을 통해 Opus 4.5로 라우팅하세요 — 공격을 스캔하고 안전한 것은 자동 승인하게 하세요 (https://code.claude.com/docs/en/hooks#permissionrequest 참고)

## 9. 데이터 및 분석에 Claude 사용하기

Claude Code에게 "bq" CLI를 사용해서 실시간으로 메트릭을 가져오고 분석하라고 하세요. 우리는 코드베이스에 BigQuery skill을 체크인해두었고, 팀의 모든 사람이 Claude Code에서 직접 분석 쿼리에 사용합니다. 개인적으로, 저는 6개월 넘게 SQL을 한 줄도 작성하지 않았습니다.

이것은 CLI, MCP, 또는 API가 있는 모든 데이터베이스에서 작동합니다.

## 10. Claude로 학습하기

팀에서 Claude Code를 학습에 사용하는 몇 가지 팁:

a. /config에서 "Explanatory" 또는 "Learning" 출력 스타일을 활성화해서 Claude가 변경 사항의 *이유*를 설명하게 하세요

b. Claude에게 익숙하지 않은 코드를 설명하는 시각적 HTML 프레젠테이션을 생성하게 하세요. 놀라울 정도로 좋은 슬라이드를 만듭니다!

c. Claude에게 새로운 프로토콜과 코드베이스를 이해하는 데 도움이 되는 ASCII 다이어그램을 그리게 하세요

d. 간격 반복 학습 skill을 만드세요: 당신이 이해한 것을 설명하면, Claude가 빈 부분을 채우기 위해 후속 질문을 하고 결과를 저장합니다
